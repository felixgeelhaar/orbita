## Orbits (Modules)

Orbita uses a modular architecture called "Orbits". Each orbit adds specialized functionality:

### Productivity Orbits
- **Smart Habits** - Adaptive habit scheduling
- **Priority Engine Pro** - Advanced prioritization algorithms
- **Focus Mode Pro** - Deep work sessions with blocking

### Scheduling Orbits

## Integration Points

### MCP (Model Context Protocol)

Orbita is AI-native with full MCP support:

```bash
# Start the MCP server
orbita mcp serve

# Connect from Claude or other AI assistants
```

With MCP, your AI assistant can:
- Create and manage tasks
- Query your schedule
- Log habit completions
- Add meeting topics
- Run reviews and planning

### API Access

Programmatic access for developers:

```bash
# Enable API mode
orbita serve --api

# API endpoints available at localhost:8080
```

### Shell Integration

Works seamlessly with your terminal workflow:

```bash
# Quick capture from anywhere
alias capture="orbita task create"

# Daily planning
alias plan="orbita schedule today && orbita task list --priority high"
```

## Architecture Highlights

### Event-Driven

- Plugin architecture for custom integrations
- MCP for AI assistants
- API for custom applications
